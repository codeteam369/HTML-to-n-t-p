Inline

Ưu điểm là tối ưu việc nắm bắt thay đổi của css, nhanh chóng khi có ít sự thay đổi
Nhược điểm:
_code sẽ lộn xộn khi có nhiều thẻ hoặc phần tử cần thay đổi
_Chỉ sửa được một cách độc lập
_Không tái sử dụng cho file html khác được

Internal được định dạng ở head bằng thẻ style

Ưu điểm

  • Giúp người code có gốc nhìn trực quan
  • Dễ định dạng lại css cho nhiều phần tử mong muốn

Nhược điểm là phải nhìn vào sự thay đổi ở nơi trên style làm cho việc nắm bắt sự thay đổi lâu hơn

External

Có ưu điểm như internal và có thể tái sử dụng file css cho các tệp html khác

Nhược điểm cũng như internal và phải tạo file riêng
